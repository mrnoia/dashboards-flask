Bootstrap Forms Overview

Bootstrap provides comprehensive form controls with styling, validation, and layout options.

Form Controls

  • .form-control: Base class for form inputs
  • .form-label: Styling for form labels
  • .form-text: Help text below inputs
  • .form-select: Styled select elements
  • .form-check: Container for checkboxes and radios
  • .form-check-input: Checkbox/radio input styling
  • .form-check-label: Checkbox/radio label styling

Input Types

  • Text inputs: text, password, email, number, tel, url
  • Textarea: Multi-line text input
  • File input: File upload controls
  • Color picker: .form-control-color
  • Range slider: Custom styled range inputs
  • Date/time: HTML5 date/time inputs

Input Groups

  • .input-group: Container for grouped inputs
  • .input-group-text: Text or icons in input groups
  • Supports prepend and append elements
  • Works with buttons, dropdowns, and text
  • Flexible sizing and positioning

Form Layout

  • Vertical forms: Default stacked layout
  • Horizontal forms: Use grid system for side-by-side
  • Inline forms: Use .row and .col for inline layout
  • Floating labels: .form-floating for modern label style
  • Grid system: Integrate with Bootstrap grid

Form Sizing

  • .form-control-lg: Large input size
  • .form-control-sm: Small input size
  • .form-select-lg/sm: Select sizing
  • Consistent sizing across all form controls
  • Responsive sizing with grid system

Form Validation

Validation States

  • .is-valid: Valid input styling
  • .is-invalid: Invalid input styling
  • .valid-feedback: Success message
  • .invalid-feedback: Error message
  • .valid-tooltip: Success tooltip
  • .invalid-tooltip: Error tooltip

Validation Methods

  • HTML5 validation: Built-in browser validation
  • Bootstrap validation: JavaScript-based validation
  • Custom validation: Custom validation logic
  • Server-side validation: Backend validation feedback

Accessibility

  • Always use labels with for attributes
  • Provide help text and instructions
  • Use proper input types and attributes
  • Ensure keyboard navigation
  • Test with screen readers
  • Use ARIA attributes when needed

Best Practices

  • Keep forms simple and focused
  • Group related fields together
  • Provide clear labels and instructions
  • Use appropriate input types
  • Implement proper validation
  • Ensure mobile-friendly design
  • Test across different devices

JavaScript Integration

  • Form validation requires Bootstrap JavaScript
  • Custom validation with JavaScript
  • Dynamic form generation
  • AJAX form submission
  • Real-time validation feedback
