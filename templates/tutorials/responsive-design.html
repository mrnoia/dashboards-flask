/* Small devices (landscape phones, 576px and up) */
@media (min-width: 576px) { ... }

@media (min-width: 768px) { ... }

@media (min-width: 1200px) { ... }

/* Extra extra large devices (larger desktops, 1400px and up) */
@media (min-width: 1400px) { ... }

Responsive Design Principles

Responsive design ensures your website looks and functions well across all devices and screen sizes.

Mobile-First Approach

  • Design for mobile devices first
  • Progressively enhance for larger screens
  • Focus on core functionality
  • Prioritize performance on mobile
  • Touch-friendly interface design

Bootstrap Breakpoints

  • xs: 0px and up (no prefix needed)
  • sm: 576px and up (small phones to tablets)
  • md: 768px and up (tablets to small desktops)
  • lg: 992px and up (desktops)
  • xl: 1200px and up (large desktops)
  • xxl: 1400px and up (extra large desktops)

Responsive Grid System

  • 12-column system: Flexible layout foundation
  • Breakpoint-specific classes: col-{breakpoint}-{number}
  • Auto-layout: Equal-width columns without numbers
  • Row columns: row-cols-{breakpoint}-{number}
  • Ordering and offsetting: Control layout flow

Responsive Utilities

Display Utilities

  • d-none: Hide element
  • d-block: Show as block
  • d-flex: Show as flex
  • Responsive: d-{breakpoint}-{value}
  • Printing: d-print-{value}

Flexbox Utilities

  • Direction: flex-direction responsive classes
  • Justify content: Horizontal alignment
  • Align items: Vertical alignment
  • Wrap: flex-wrap responsive behavior

Text Utilities

  • Alignment: text-{breakpoint}-{alignment}
  • Transform: text-{breakpoint}-{transform}
  • Weight and italics: Responsive typography
  • Visibility: Responsive text visibility

Responsive Media

Images

  • .img-fluid: Max-width: 100%, height: auto
  • .img-thumbnail: Responsive with padding and border
  • Figure elements: Semantic image containers
  • Responsive images: srcset and sizes attributes

Tables

  • .table-responsive: Horizontal scrolling on small screens
  • Breakpoint-specific: .table-responsive-{breakpoint}
  • Stacked tables: Custom responsive table layouts

Responsive Navigation

  • Collapsible navbars: Mobile hamburger menu
  • Dropdown menus: Space-saving navigation
  • Tabs and pills: Responsive tab navigation
  • Breadcrumbs: Hierarchical navigation

Best Practices

  • Start with mobile design first
  • Use relative units (%, em, rem, vh, vw)
  • Optimize images for different screen sizes
  • Test on actual devices, not just emulators
  • Consider touch interactions on mobile
  • Prioritize content hierarchy
  • Ensure accessibility across all screen sizes

Performance Considerations

  • Optimize images for mobile devices
  • Minimize HTTP requests
  • Use responsive images with srcset
  • Load critical CSS first
  • Consider lazy loading for images
  • Test performance on slow connections

Testing Responsive Design

  • Browser developer tools device emulation
  • Real device testing
  • Viewport testing at all breakpoints
  • Touch interaction testing
  • Performance testing on mobile
  • Cross-browser compatibility testing
